## Project Overview

This project provides a complete, reproducible Stata workflow for analysing individual-level labour market panel data. It replicates the core methods from the **Mincer (1974) earnings equation** and extends them to address endogeneity using panel fixed effects and first-difference estimators.

The dataset is a **balanced panel**: 20 individuals √ó 8 years = **160 person-year observations**, with planned missingness in wages during unemployment spells (consistent with real-world survey design).

## Research Questions

| # | Research Question | Method |
|---|---|---|
| 1 | What are the returns to an additional year of schooling? | OLS / FE Mincer equation |
| 2 | Is there a union wage premium after controlling for selection? | FE / FD estimation |
| 3 | How large is the conditional gender wage gap? | OLS with controls |
| 4 | Do fixed effects and random effects yield different results? | Hausman specification test |
| 5 | Did the 2020 COVID-19 shock reduce employment probability? | FE linear probability model |
| 6 | Does the return to experience diminish over the career? | Quadratic experience term |

## Data Management

### Load and declare panel structure

```stata
import delimited "data/labor_panel.csv", clear varnames(1)

* Declare as panel data
xtset person_id year, yearly
xtdescribe
```

### Handle missing wages (unemployment spells)

```stata
* ln_wage is system missing (.) for unemployed observations
* Use if-conditions to restrict wage regressions to employed spells:
summarize ln_wage if employed == 1
```

### Create factor variables for categorical regressors

```stata
* Stata encodes string categoricals as factors using i. prefix
* Always check encoding before regression:
tab gender,   generate(female_)
tab race,     generate(race_)
tab industry, generate(ind_)

* Or use directly in regress:
reg ln_wage i.gender i.race i.industry i.region, robust
```

### Encode string variables to numeric (required for xt commands)

```stata
encode gender,        generate(gender_n)
encode race,          generate(race_n)
encode region,        generate(region_n)
encode industry,      generate(industry_n)
encode occupation,    generate(occ_n)
encode firm_size,     generate(firmsize_n)
encode health_status, generate(health_n)
```

## Econometric Methods

### 1. Mincer Earnings Equation (OLS)

The workhorse model for wage determination:

**ln(w·µ¢‚Çú) = Œ≤‚ÇÄ + Œ≤‚ÇÅ ¬∑ S·µ¢ + Œ≤‚ÇÇ ¬∑ Exp·µ¢‚Çú + Œ≤‚ÇÉ ¬∑ Exp·µ¢‚Çú¬≤ + X·µ¢‚Çú'Œ≥ + Œµ·µ¢‚Çú**

Where *S* = years of schooling, *Exp* = potential experience.

```stata
reg ln_wage education_years experience experience_sq ///
    i.gender i.race i.region i.industry ///
    union_member tenure_years urban married children ///
    if employed==1, robust

* Return to schooling (approx. %)
di "Return to 1 year of schooling: " _b[education_years]*100 "%"
```

### 2. Random Effects (RE) ‚Äî `xtreg, re`

Assumes individual-specific effects are **uncorrelated** with regressors. Efficient but inconsistent if this assumption fails.

```stata
xtreg ln_wage education_years experience experience_sq ///
    union_member tenure_years married urban ///
    if employed==1, re
```

### 3. Fixed Effects (FE) ‚Äî `xtreg, fe`

Eliminates all **time-invariant** unobservables (ability, personality, stable preferences). Time-invariant regressors (education, gender, race) are absorbed and cannot be estimated.

```stata
xtreg ln_wage experience experience_sq ///
    union_member tenure_years married urban ///
    if employed==1, fe robust
```

### 4. Hausman Test ‚Äî FE vs RE

```stata
* Step 1: Store FE estimates
xtreg ln_wage experience experience_sq union_member tenure_years ///
    if employed==1, fe
estimates store fe

* Step 2: Store RE estimates
xtreg ln_wage experience experience_sq union_member tenure_years ///
    if employed==1, re
estimates store re

* Step 3: Test
hausman fe re
* p < 0.05 ‚Üí reject RE ‚Üí use Fixed Effects
```

### 5. First-Difference Estimator ‚Äî `reg` on Œî variables

Equivalent to FE with T=2; more transparent for long panels.

```stata
sort person_id year
gen d_ln_wage    = d.ln_wage
gen d_experience = d.experience
gen d_union      = d.union_member

reg d_ln_wage d_experience d_union if employed==1, ///
    robust cluster(person_id)
```

### 6. Estimator Comparison

| Estimator | Handles unobservables? | Time-invariant regressors? | Efficiency |
|---|---|---|---|
| Pooled OLS | No | Yes | Highest (if correct) |
| Random Effects | Partially (GLS) | Yes | High |
| Fixed Effects | Yes (time-invariant) | No | Lower |
| First Difference | Yes (time-invariant) | No | Lower |

## Common Pitfalls

| Pitfall | Consequence | Fix |
|---|---|---|
| Forgetting `xtset` before `xtreg` | Stata throws an error | Always `xtset` before any panel command |
| Including time-invariant vars in FE | They drop silently (absorbed) | Use RE or between estimator for time-invariant effects |
| Using `robust` alone in panel data | Underestimates standard errors | Use `vce(cluster person_id)` to account for serial correlation |
| Interpreting Œ≤ on log-wage directly | Overstates the effect for large coefficients | Use `(exp(Œ≤)-1)*100` for percentage interpretation |
| Running Hausman without storing estimates first | Hausman fails | Use `estimates store` after each `xtreg` |
| Including levels in FD regression | First-differencing removes the constant | Never include untransformed level variables in FD models |
| Comparing R¬≤ across OLS / RE / FE | R¬≤ is not comparable across estimators | Report R¬≤-within and R¬≤-between separately for panel estimators |
